# Handoff Folder

This folder is the shared coordination space between Codex and Gemini.

## Purpose

- Store handoff notes
- Record current plans
- Track open blockers
- Capture interface contracts that affect both agents

## Files

- `current_plan.md`: active repo-level execution plan
- `codex_handoff.md`: Codex-owned notes, backend updates, and integration guidance
- `gemini_handoff.md`: Gemini-owned notes, UI updates, and frontend presentation guidance
- `shared_decisions.md`: decisions that both agents should follow

## Read Order

1. `AGENTS.md`
2. `GEMINI.md`
3. `handoff/README.md`
4. `handoff/current_plan.md`
5. `handoff/shared_decisions.md`

## Rules

- Keep notes concise and current.
- Do not use this folder for generated noise.
- Add handoff context here when one agent's work changes the expectations for the other.
- Prefer updating existing files over creating many small one-off docs.
